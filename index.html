Git .gitignore使用

发表于 2018-03-29 | 更新于 2018-09-11 | 分类于 维护

.gitignore

  • Location: 项目根目录，与.git在同一层

  • 匹配规则：

    • 使用glob模式
    • .gitignore每一行都是文件名匹配规则
    • 可以参考下gitignore 各语言规则

  • .gitignore应该在项目创建的时候创建，如果有些文件已经被收入了VCS中（已经被追踪并上传到github），新的匹配规则对已经上传的文件是无效的。如果希望新的匹配规则可以覆盖整个项目，让已经上传的文件被忽略，则需要删除git的本地缓存（让已上传的文件处于未追踪状态），再提交
