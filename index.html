Java 基础——I/O

发表于 2019-09-10 | 更新于 2019-11-02

Java 工程师说起 IO，基本都会脱口而出三个词： BIO – Blocking IO，NIO – Non-Blocking IO，AIO – Asynchronous IO。如果表示不知道，恭喜你，你还没被洗脑。很多人在讨论这三个术语时，会把 NIO 当成 java.nio 也就是 “New IO”，只能说你可能真有点萌，再如何 oracle 也没实现 java.bio 和 java.aio。这些术语并不是 Java 专有的，因为他们对标的是 操作系统的 IO 模型，本文尽可能略过，有兴趣自阅。总的来说，各语言对操作系统的 IO 模型进行封装和实现，归结起来就是这三种。

IO I/O（下文可能为了方便还是会写成IO），指的是各种数据源之间的数据交换。

思考一个问题：内存块之间的数据交换是 IO 吗？在我看来，如果此时把数据源的参照物当成是内存块，那么他们之间的数据交换是 IO，而如果把参照物认为是程序，那么这不算IO。

阅读 Java IO 相关的文档就会发现，Java 的 I/O 更多是以程序本身为数据源的参照物，想方设法与外部交互。Java 的基本 I/O 是流式 I/O，即 I/O Streams。

Git .gitignore使用

发表于 2018-03-29 | 更新于 2018-09-11 | 分类于 维护

.gitignore

  • Location: 项目根目录，与.git在同一层

  • 匹配规则：

    • 使用glob模式
    • .gitignore每一行都是文件名匹配规则
    • 可以参考下gitignore 各语言规则

  • .gitignore应该在项目创建的时候创建，如果有些文件已经被收入了VCS中（已经被追踪并上传到github），新的匹配规则对已经上传的文件是无效的。如果希望新的匹配规则可以覆盖整个项目，让已经上传的文件被忽略，则需要删除git的本地缓存（让已上传的文件处于未追踪状态），再提交
